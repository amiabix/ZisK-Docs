<Summary>

`zisklib` functions for secp256k1 (K-256) base and scalar field arithmetic,
curve point operations, ECDSA signature verification and recovery, and
BIP-340 Schnorr verification.

</Summary>

## Module overview

secp256k1 is the curve underlying Bitcoin, Ethereum, and most modern
blockchains. Chain-validation guests touch every layer of the stack, so
the zisklib surface exposes the full ECC hierarchy rather than only the
top-level signature primitives.

The base field reduces hashes and encodes coordinates. The scalar field
holds private keys, ECDSA nonces, and `s` values. Curve points carry
public keys and intermediate sums. Signatures sit on top: ECDSA backs
EVM transaction validation, Schnorr backs BIP-340 taproot.

## Hierarchy

Four layers stack onto each other. Each level is a strict client of the
ones below it.

| Layer | What it is | Used for |
|---|---|---|
| Fp (base field) | Arithmetic mod the prime `p` | Coordinate arithmetic, hash-to-curve |
| Fn (scalar field) | Arithmetic mod the curve order `n` | Private keys, nonces, ECDSA `s` |
| Curve | Affine points on `y² = x³ + 7` | Public keys, point sums, scalar mul |
| Signatures | ECDSA verify/recover, Schnorr verify | Transaction validation |

:::note
Points are `[u64; 8]` (x ++ y, 4 LE limbs each, 256 bits per coordinate).
Field elements (Fp) and scalar elements (Fn) are `[u64; 4]` LE limbs.
The `hints: &mut Vec<u64>` argument shown in each signature is gated by
`#[cfg(feature = "hints")]` and is only present when that feature is
enabled.
:::

## Base field (Fp)

Arithmetic modulo the secp256k1 prime `p`. Callers reduce raw 256-bit
inputs, then add, multiply, square, invert, or take square roots.
Coordinate arithmetic in the curve layer and the `r ≡ x (mod p)` check
in ECDSA both bottom out here.

<FnDef name="sqrt_fp_secp256k1">

Square root in the secp256k1 base field Fp. Returns `(sqrt, is_qr)`.
When `is_qr` is true, `sqrt` satisfies `sqrt² ≡ x (mod p)` with the
requested parity. When `is_qr` is false, `sqrt` satisfies
`sqrt² ≡ x·NQR (mod p)` for the implementation-defined non-residue `NQR`.

```rust
pub fn sqrt_fp_secp256k1(
    x: &[u64; 4],
    parity: u64,
    #[cfg(feature = "hints")] hints: &mut Vec<u64>,
) -> ([u64; 4], bool)
```

#### Parameters

| Name | Type | Description |
|---|---|---|
| `x` | `&[u64; 4]` | Fp element to take the square root of. |
| `parity` | `u64` | Selects the root: `0` for even, `1` for odd canonical representative. |
| `hints` | `&mut Vec<u64>` | Hint buffer (only present with `hints` feature). |

#### Returns

| Type | Description |
|---|---|
| `([u64; 4], bool)` | The square root and a flag indicating whether `x` is a quadratic residue. |

#### Example

```rust
let (root, is_qr) = zisklib::sqrt_fp_secp256k1(&x_limbs, 0);
if is_qr {
    // root is a valid square root of x mod p
}
```

</FnDef>

## Scalar field (Fn)

Arithmetic modulo the curve order `n`. Private keys, ECDSA nonces, and
the `s` value of a signature live here. The curve layer reduces scalars
through this field before each scalar multiplication, and ECDSA verify
performs its `s⁻¹` and `z·s⁻¹` work directly on Fn elements.

## Curve

Point addition, doubling, and scalar multiplication on `y² = x³ + 7`.
Built on Fp for coordinate arithmetic and Fn for scalar reduction.
Used directly by ECDSA `recover` to compute `s⁻¹·(z·G + r·R)` and by
Schnorr `verify` to check `s·G - e·P == R`. Multi-scalar variants back
batch verification.

| `Option<[u64; 8]>` | `Some(Σ kᵢ·Pᵢ)`, or `None` if the result is the point at infinity. |

#### Example

```rust
let result = zisklib::multi_scalar_mul_secp256k1(&scalars, &points);
```

</FnDef>

## ECDSA

ECDSA signature verification and public-key recovery. Verify checks an
existing `(r, s)` against a known public key. Recover reconstructs the
public key from `(r, s, z, recid)`, the path Ethereum uses to derive
the signer address from a transaction. Both pull from Fn for inversion
and from the curve layer for the joint scalar multiplication.

<FnPanel>

<FnDef name="ecdsa_verify_secp256k1">

Verifies an ECDSA signature `(r, s)` over message hash `z` with public
key `pk`. Internally hints `(x, y) = z·s⁻¹·G + r·s⁻¹·pk`, checks that
`[z]G + [r]pk + [-s](x, y) == 𝒪`, and that `x ≡ r (mod n)`.

```rust
pub fn ecdsa_verify_secp256k1(
    pk: &[u64; 8],
    z: &[u64; 4],
    r: &[u64; 4],
    s: &[u64; 4],
    #[cfg(feature = "hints")] hints: &mut Vec<u64>,
) -> bool
```

#### Parameters

| Name | Type | Description |
|---|---|---|
| `pk` | `&[u64; 8]` | Public key as an affine point (x ++ y limbs). |
| `z` | `&[u64; 4]` | Message hash as 4 LE limbs. |
| `r` | `&[u64; 4]` | Signature `r` component as 4 LE limbs. |
| `s` | `&[u64; 4]` | Signature `s` component as 4 LE limbs. |
| `hints` | `&mut Vec<u64>` | Hint buffer (only present with `hints` feature). |

#### Returns

| Type | Description |
|---|---|
| `bool` | `true` if the signature is valid for the given public key and message hash, `false` otherwise. |

#### Example

```rust
let valid = zisklib::ecdsa_verify_secp256k1(&pk, &msg_hash, &r, &s);
assert!(valid);
```

</FnDef>

<FnDef name="ecdsa_recover_secp256k1">

Recovers the public-key point from a compact signature `(r, s)`, message
hash `z`, and recovery id `recid` (0 or 1). On failure returns one of the
error codes `ECDSA_RECOVER_ERR_INVALID_R` (1), `ECDSA_RECOVER_ERR_INVALID_S`
(2), `ECDSA_RECOVER_ERR_INVALID_RECID` (3), `ECDSA_RECOVER_ERR_POINT_NOT_ON_CURVE`
(4), or `ECDSA_RECOVER_ERR_RECOVERY_FAILED` (5).

```rust
pub fn ecdsa_recover_secp256k1(
    r: &[u64; 4],
    s: &[u64; 4],
    z: &[u64; 4],
    recid: u8,
    #[cfg(feature = "hints")] hints: &mut Vec<u64>,
) -> Result<[u64; 8], u8>
```

#### Parameters

| Name | Type | Description |
|---|---|---|
| `r` | `&[u64; 4]` | Signature `r` component as 4 LE limbs. |
| `s` | `&[u64; 4]` | Signature `s` component as 4 LE limbs. |
| `z` | `&[u64; 4]` | Message hash as 4 LE limbs. |
| `recid` | `u8` | Recovery id: `0` or `1`. |
| `hints` | `&mut Vec<u64>` | Hint buffer (only present with `hints` feature). |

#### Returns

| Type | Description |
|---|---|
| `Result<[u64; 8], u8>` | `Ok([u64; 8])` with the recovered public key as an affine point, or `Err(code)` with one of the `ECDSA_RECOVER_ERR_*` codes. |

#### Example

```rust
let pk = zisklib::ecdsa_recover_secp256k1(&r, &s, &z, 0)
    .expect("public key recovery failed");
```

</FnDef>

</FnPanel>

## Schnorr (BIP-340)

BIP-340 Schnorr verification, the signature scheme used by Bitcoin
taproot. Inputs are 32-byte big-endian, x-only public keys. The single
verifier checks one signature; the batch verifier folds many into a
single Pippenger MSM through the curve layer for amortized cost.

<FnPanel>

<FnDef name="schnorr_verify_secp256k1">

Verifies a single BIP-340 Schnorr signature on secp256k1 over an
arbitrary-length message. Public key, `r`, and `s` are 32-byte
big-endian inputs.

```rust
pub fn schnorr_verify_secp256k1(
    msg: &[u8],
    pk_x: &[u8; 32],
    sig_r: &[u8; 32],
    sig_s: &[u8; 32],
    #[cfg(feature = "hints")] hints: &mut Vec<u64>,
) -> bool
```

#### Parameters

| Name | Type | Description |
|---|---|---|
| `msg` | `&[u8]` | Arbitrary-length message bytes. |
| `pk_x` | `&[u8; 32]` | x-only public key, 32 bytes big-endian. |
| `sig_r` | `&[u8; 32]` | Signature `r` component, 32 bytes big-endian. |
| `sig_s` | `&[u8; 32]` | Signature `s` component, 32 bytes big-endian. |
| `hints` | `&mut Vec<u64>` | Hint buffer (only present with `hints` feature). |

#### Returns

| Type | Description |
|---|---|
| `bool` | `true` if the signature is valid per BIP-340, `false` otherwise. |

#### Example

```rust
let valid = zisklib::schnorr_verify_secp256k1(msg, &pk_x, &sig_r, &sig_s);
assert!(valid);
```

</FnDef>

<FnDef name="schnorr_batch_verify_secp256k1">

BIP-340 batch verification using a single Pippenger multi-scalar
multiplication. The four input slices must all have length `u`. Panics
if the slice lengths disagree. Returns `true` only when every signature

```rust

    pk_xs: &[&[u8; 32]],
    sig_rs: &[&[u8; 32]],
    sig_ss: &[&[u8; 32]],
    #[cfg(feature = "hints")] hints: &mut Vec<u64>,
) -> bool
```

#### Parameters

| Name | Type | Description |
|---|---|---|
| `msgs` | `&[&[u8]]` | Per-signature message slices. |
| `pk_xs` | `&[&[u8; 32]]` | Per-signature x-only public keys, big-endian. |
| `sig_rs` | `&[&[u8; 32]]` | Per-signature `r` components, big-endian. |
| `sig_ss` | `&[&[u8; 32]]` | Per-signature `s` components, big-endian. |
| `hints` | `&mut Vec<u64>` | Hint buffer (only present with `hints` feature). |

#### Returns

| Type | Description |
|---|---|
| `bool` | `true` if every signature in the batch verifies, `false` otherwise. |

#### Example

```rust
let valid = zisklib::schnorr_batch_verify_secp256k1(
    &msgs, &pk_xs, &sig_rs, &sig_ss,
);
assert!(valid);
```
